{ No. 3 | Process }

Research

{Open up chrome and dust off the moleskin.}

​

This step is about recognizing problems, defining goals and developing a plan of action. It's about learning the user's needs but also determining constraints. It's also important to learn about the competition and determine how to improve what is already available to the user.

Wireframe

{It's all about the flow.}

​

It's easy for a project to get confusing. Team members come from different backgrounds and the vision is up for interpretation. Wireframing helps keep all the stakeholders on the same page. They allow for planning the user flow while giving everyone a visual of the project plan.

Design

{Make it shine.}

​

And make it pretty. Users expect an interface that is intuitive, appealing and fun to use. Also make it make sense because it's about more than a pretty picture.

Develop

{Make it work.}

​

These days interfaces need to work across multiple devices. Building, testing and tweaking are part of the game. Make sure the interactions are user friendly and intuitive.
